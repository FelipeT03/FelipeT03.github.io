Resultados del Modelo BiLSTM NER

Conjunto de datos: CoNLL-2002 (Español)

Los modelos fueron entrenados y evaluados utilizando el conjunto de datos CoNLL-2002 para español. Este conjunto de datos está específicamente diseñado para tareas de Reconocimiento de Entidades Nombradas (NER) y contiene texto anotado con varios tipos de entidades, incluyendo nombres de personas, en los cuales nos enfocamos para este proyecto.

¿Qué es un Modelo BiLSTM?

Un modelo BiLSTM (Memoria a Corto-Largo Plazo Bidireccional) es un tipo de red neuronal recurrente que procesa secuencias de entrada en ambas direcciones, hacia adelante y hacia atrás. Este enfoque bidireccional permite al modelo capturar el contexto tanto de elementos pasados como futuros en una secuencia, haciéndolo particularmente efectivo para tareas como el Reconocimiento de Entidades Nombradas.

Opción 1: Modelo BiLSTM a nivel de token

En este enfoque, el modelo predice si cada token individual en una frase es un elemento de datos personales (específicamente, un nombre de persona).

Beneficios de la predicción a nivel de token:

  • Identificación detallada: Puede señalar exactamente qué palabras en una frase son datos personales.
  • Consciente del contexto: La arquitectura BiLSTM permite al modelo considerar las palabras circundantes al hacer predicciones, mejorando la precisión.
  • Flexible: Puede manejar frases de diferentes longitudes e identificar múltiples elementos de datos personales en una sola frase.

Resultados:

Métrica Valor
Exactitud 0.985
Precisión 0.856
Exhaustividad 0.871
Puntuación F1 0.863

Opción 2: Modelo BiLSTM a nivel de frase

Este enfoque predice si una frase en su totalidad contiene algún dato personal (nombres de personas), sin especificar qué tokens son los datos personales.

Resultados:

Métrica Valor
Exactitud 0.912
Precisión 0.897
Exhaustividad 0.925
Puntuación F1 0.911

Conclusión

Ambos modelos muestran un fuerte rendimiento en la identificación de datos personales, con el modelo a nivel de token (Opción 1) proporcionando información más detallada sobre la ubicación de los datos personales dentro de las frases. El modelo a nivel de frase (Opción 2) muestra métricas generales ligeramente más altas, posiblemente debido a la tarea de clasificación binaria más simple a nivel de frase.

La elección entre estos modelos dependería del caso de uso específico: si se requiere una identificación precisa de los elementos de datos personales, la Opción 1 sería preferible. Si solo se necesita una clasificación binaria de frases que contienen datos personales, la Opción 2 podría ser más adecuada.
